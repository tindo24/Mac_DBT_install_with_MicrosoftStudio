Azure Data Studio Replaces SSMS on macOS

SSMS (SQL Server Management Studio) is Windows-only. Microsoft's official Mac equivalent is Azure Data Studio — it provides full SQL Server connectivity, query editing, and object browsing.

Go to the Azure Data Studio download page at the link below.
Choose your chip: ARM64 for Apple Silicon (M1/M2/M3/M4) or x64 for Intel Mac.
Extract the downloaded .zip and drag Azure Data Studio.app to your /Applications folder.
Launch the app. If macOS blocks it, go to System Settings → Privacy & Security → Open Anyway.
Click New Connection, enter your SQL Server hostname, and authenticate.
DOWNLOAD URL
https://learn.microsoft.com/en-us/azure-data-studio/download-azure-data-studio
⚠ macOS Security If you see "developer cannot be verified", go to System Settings → Privacy & Security → Open Anyway.
2
Visual Studio Code Recommended code editor for dbt

VS Code is the go-to editor for dbt development, with excellent extensions for SQL, Python, YAML, and dbt-specific syntax highlighting.

Download VS Code for macOS from code.visualstudio.com/download.
Unzip and move Visual Studio Code.app to /Applications.
Open VS Code and install these extensions via the Extensions panel (Cmd+Shift+X).
Add VS Code to PATH: press Cmd+Shift+P, type "Shell Command", select Install 'code' command in PATH.
ExtensionPurpose
dbt Power UserAutocomplete, previews, lineage graphs
PythonPython language support & debugging
SQL ToolsSQL query execution and formatting
YAMLSyntax highlighting for dbt schema files
3
Python 3.7 or Above Runtime required by dbt

dbt-core requires Python 3.7+. macOS may have Python 2.7 or nothing installed — always install Python 3 explicitly.

TERMINAL — Install Homebrew (if needed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
TERMINAL — Install & verify Python
brew install python python3 --version # Expected: Python 3.x.x (must be 3.7 or higher)
ℹ Recommended: Use a Virtual Environment python3 -m venv dbt-env
source dbt-env/bin/activate Activate this environment before running any pip install commands.
4
pip install dbt-core Core dbt framework

dbt-core provides the CLI, project management, and model compilation logic. Install it after Python is set up.

TERMINAL
pip install dbt-core # If permission errors occur: pip install --user dbt-core # Verify installation: dbt --version
⚠ pip vs pip3 If both Python 2 and 3 are installed, use pip3 to ensure packages install for the correct version.
5
pip install dbt-(adapter) Database-specific connector

dbt-core alone can't connect to any database. Install the adapter that matches your database platform.

DatabaseInstall Command
SQL Server / Azure Synapsepip install dbt-sqlserver
Snowflakepip install dbt-snowflake
PostgreSQLpip install dbt-postgres
BigQuerypip install dbt-bigquery
Redshiftpip install dbt-redshift
Databrickspip install dbt-databricks
DuckDBpip install dbt-duckdb
TERMINAL — Example: SQL Server
pip install dbt-sqlserver dbt --version # Adapter should now appear under installed adapters
6
ODBC Driver Local database connectivity layer

An ODBC driver is the low-level connectivity layer between dbt/Python and your database. For SQL Server on Mac, install Microsoft's driver via Homebrew.

TERMINAL — SQL Server ODBC Driver (via Homebrew)
# Add Microsoft tap brew tap microsoft/mssql-release https://github.com/Microsoft/homebrew-mssql-release # Install unixODBC and the SQL Server driver brew install unixodbc HOMEBREW_ACCEPT_EULA=Y brew install msodbcsql18 # Optional: install sqlcmd command-line tools HOMEBREW_ACCEPT_EULA=Y brew install mssql-tools18 # Verify the driver is registered odbcinst -q -d
DatabaseODBC Notes
SQL ServerInstall msodbcsql18 via Homebrew (above)
SnowflakeNo ODBC needed — uses native Python connector
PostgreSQLNo ODBC needed — uses psycopg2 (bundled with dbt-postgres)
BigQueryNo ODBC needed — uses google-cloud-bigquery
DatabricksDownload Simba Spark ODBC Driver from Databricks website

✅ Final Verification

Once all steps are complete, run these commands to confirm your full setup is working:

TERMINAL
dbt --version python3 --version odbcinst -q -d # SQL Server only # Start a new dbt project: dbt init my_project

🎉 You're ready to build your first dbt project!
